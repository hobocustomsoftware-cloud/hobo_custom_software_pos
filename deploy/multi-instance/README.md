# Scalable Multi-Client POS (Myanmar + English)

Deploy **multiple isolated POS instances** (one per shop) on a single server. Bilingual: Myanmar Unicode + English; Noto Sans Myanmar for UI and PDF/receipts.

Each client has:

- **Separate containers**: `pos_app_${CLIENT_NAME}`, `pos_db_${CLIENT_NAME}`, `pos_redis_${CLIENT_NAME}`
- **Separate database**: `pos_db_${CLIENT_NAME}` (100% data isolation)
- **Resource limits**: 256MB RAM per container (app 256MB, postgres 256MB, redis 64MB) so you can host 20+ clients on one server
- **30-day trial**: After 30 days from shop creation, POS is locked and shows "Trial Expired" with your contact info until activated

## Quick start

From the **repository root**:

```bash
./deploy/multi-instance/setup_client.sh SHOP_NAME PORT
```

Example:

```bash
./deploy/multi-instance/setup_client.sh myshop 8001
./deploy/multi-instance/setup_client.sh beta    8002
```

This creates:

- `deploy/multi-instance/clients/acme/` with `.env` and `docker-compose.yml`
- Starts containers with project name `acme` (volumes prefixed by project name)
- App at **http://localhost:8001/app/** (register, then use POS)

## Files

| File | Purpose |
|------|--------|
| `docker-compose.template.yml` | Template: postgres + backend + redis, env-substituted |
| `setup_client.sh` | Creates client dir, generates SECRET_KEY/DB_PASSWORD, runs `docker compose -p CLIENT_NAME up -d` |
| `nginx-multi-domain.conf.template` | Nginx: map `clientname.yourdomain.com` → `http://127.0.0.1:PORT` |
| `clients/<name>/` | Per-client `.env` and `docker-compose.yml` (do not commit `.env`) |

## Trial and lock (Django)

- **Model**: `core.ShopSettings.trial_start_date` (set on first `get_settings()` create).
- **Middleware**: `core.trial_middleware.TrialExpiryMiddleware` — if `now > trial_start_date + 30 days`, any API under `/api/` except login/register/license/shop-settings returns:

  **403** with `error`, `message`, and `contact` (set via `TRIAL_EXPIRED_CONTACT` in client .env). Frontend shows `/trial-expired` page with your contact (Myanmar + English).

- To “activate” a client: set `trial_start_date` to `null` in Django admin or DB (or extend the date).

## Nginx (subdomain → Docker port)

Link subdomains like **https://client.domain.com** to the correct Docker port:

1. Copy `nginx-multi-domain.conf.template` and replace:
   - `CLIENT_SUBDOMAIN` → e.g. `myshop`
   - `PORT` → e.g. `8001`
   - `yourdomain.com` → your domain
2. Add one server block per client (duplicate the block for each shop).
3. Reload Nginx: `sudo nginx -t && sudo systemctl reload nginx`.

Result: `https://myshop.yourdomain.com` → `http://127.0.0.1:8001`.

## Standalone focus

- Each client uses **its own PostgreSQL database** (`pos_db_${CLIENT_NAME}`) and own volumes; no data is shared between clients.
- Each instance has its own `SECRET_KEY` and `DB_PASSWORD` (generated by `setup_client.sh`).

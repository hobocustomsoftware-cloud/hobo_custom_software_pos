# 404 / 401 စစ်ဆေးချက် နှင့် ကျန်နေနိုင်သော အချက်များ

**နောက်ဆုံး စစ်ဆေးချက်:** 2026-02-21

---

## ပြီးသွားပြီး (လုပ်ထားပြီး)

### 404 ကာကွယ်ခြင်း
| အချက် | လုပ်ချက် |
|--------|-----------|
| **shop-settings** | Backend မှာ `re_path(r'^api/core/shop-settings/?$', ...)` ထည့်ပြီး၊ view မှာ error ဖြစ်ရင် default JSON ပြန်အောင် လုပ်ထားသည်။ |
| **Frontend** | shop-settings 404/မရှိရင် `apiUnavailable` သတ်မှတ်ပြီး Login မှာ "Backend စတင်ထားပါ" သတိပေးချက် ပြသည်။ |

### 401 ကာကွယ်ခြင်း
| အချက် | လုပ်ချက် |
|--------|-----------|
| **offlinePos** | Login မဝင်ရသေးချိန်မှာ staff/items မခေါ်အောင် `_hasToken()` စစ်ပြီး `fetchProductsAndCache` / `syncPendingSales` ကို token ရှိမှသာ ခေါ်သည်။ |
| **Login မက်ဆေ့ချ်** | 401 + "No active account" ဖြစ်ရင် "ဤအကောင့်မရှိပါ... စာရင်းသွင်းရန် နှိပ်ပါ" နဲ့ Register လင့်ပြသည်။ |
| **Backend မရှိ** | Login/Register မှာ network/502 ဖြစ်ရင် "Backend ချိတ်ဆက်မှု မရပါ" ပြသည်။ |

### API Interceptor
- 403 `license_expired` ဖြစ်ရင် `/license-activate` သို့ ပြောင်းသည်။
- Token ရှိရင် request တိုင်းမှာ `Authorization: Bearer` ထည့်သည်။

## ကျန်နေနိုင်သော 404 (စစ်သင့်သည်များ)

### ၁။ Backend မစတင်ထားခြင်း
- **လက္ခဏာ:** shop-settings, token, register စသည်အားလုံး 404 သို့မဟုတ် 502 ရနိုင်သည်။
- **ဖြေရှင်းနည်း:** Backend စတင်ပါ – `python manage.py runserver 0.0.0.0:8000` (သို့) `docker-compose up -d backend`။

### ၂။ Frontend path မှား (leading slash) – ပြင်ပြီး
- API path များကို leading slash မပါအောင် ပြင်ထားပြီး (ဥပမာ `'service/track/'`, `'core/reset-password/'`, `'invoices/'` စသည်)။ baseURL `/api/` နဲ့ ပေါင်းရင် `/api/service/track/` စသည်ဖြစ်မည်။
- **ပြင်ထားသော ဖိုင်များ:** RepairTrack.vue, ResetPassword.vue, WarrantyCheck.vue, SalesReport.vue, ServiceReport.vue, ForgotPassword.vue, InventoryReport.vue, CustomerReport.vue။

### ၃။ Backend route မရှိသေး
- Frontend ခေါ်တဲ့ endpoint တစ်ခုခုက Backend မှာ မထည့်ထားရင် 404 ပြန်မယ်။
- လက်ရှိ စစ်ထားသည်များမှာ core, inventory, service, customer, accounting, installation, license, ai URL များ ကိုက်ညီသည်။

## ကျန်နေနိုင်သော 401

### ၁။ Token မရှိခြင်း ( login ထားခြင်း)
- **လက္ခဏာ:** Login မဝင်ပဲ protected page (သို့) API ခေါ်ရင် 401 ရမယ်။
- **ဖြေရှင်း:** Router က public route မဟုတ်ရင် login သို့ ပြောင်းပြီးသား။ API ခေါ်တိုင်း token ထည့်ပြီးသား (api.js interceptor)။

### ၂။ Token သက်တမ်းကုန်ခြင်း
- **လက္ခဏာ:** ဝင်ထားပြီး ကြာမှ 401 ရမယ်။
- **ဖြေရှင်း:** auth store မှာ `core/me/` 401 ရရင် logout လုပ်ပြီးသား။ လိုရင် refresh token သုံးပြီး token သစ်ယူခြင်း ထည့်နိုင်သည်။

### ၃။ အကောင့်မရှိခြင်း (စာရင်းမသွင်းရသေး)
- **လက္ခဏာ:** Login မှာ username/password ထည့်ပြီး 401 "No active account found" ရမယ်။
- **ဖြေရှင်း:** Login မှာ ရှင်းလင်းသော မက်ဆေ့ချ် နဲ့ "စာရင်းသွင်းရန်" လင့် ပြထားပြီး။ Register ကနေ ပထမဦးဆုံး user စာရင်းသွင်းပြီး ပြန် login ဝင်ပါ။

## အတိုချုပ်

| အမျိုးအစား | လုပ်ပြီး | ကျန်နေနိုင် |
|---------------|----------|----------------|
| **404** | shop-settings route + default response, apiUnavailable + သတိပေးချက် | Backend မဖွင့်ရင် 502/404၊ path မှားရင် 404 |
| **401** | offlinePos token စစ်ခြင်း, Login မက်ဆေ့ချ်, 403 license redirect | Token သက်တမ်းကုန်ရင် refresh ထည့်ချင်ရင် ထည့်နိုင် |

### စစ်ဆေးရန် အဆင့်များ
1. **Backend စတင်ပါ** – 404/502 များစွာ ပျောက်သင့်သည်။  
2. **ပထမဦးဆုံး user** – Register ကနေ စာရင်းသွင်းပြီး ထို user နဲ့ Login ဝင်ပါ (401 "No active account" မရအောင်)။  
3. **Console/Network** – 404 ရတဲ့ request URL ကို ကြည့်ပြီး Backend မှာ route ရှိ/မရှိ စစ်ပါ။  

အသေးစိတ် shop-settings 404 ဖြေရှင်းနည်း: `docs/TROUBLESHOOTING_404_SHOP_SETTINGS.md`။
